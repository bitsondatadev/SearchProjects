href="/report/">Corrections &amp; Updates</a> </li> </ul> </div> <div id="feature"> <span class="feature_title">Policy on UNIX <code>find</code> Command</span> <p>Users should <em>never</em>, <em>ever</em> run the <code>find</code> command starting at <code>/</code> under any circumstances. If you think you have a reason, contact the <a href="mailto:software@cs.purdue.edu">facilities software staff</a>. In every case I have seen there are more efficient ways to do what you want. In fact, you should never run <code>find</code> to search any directories except your own.</p> <p> If you are looking for software, you should check out the <a href="http://support.cs.purdue.edu/help/Software,_UNIX">Software</a> page or use the <code>trouble</code> command to send a request for information to the staff. We will tell you where it is if we have it installed. If it isn't installed, you won't find it anyway.</p> <p>Using <code>find /</code> puts a tremendous load on the networks and all machines in the department. It not only takes forever (and almost never does what you want anyway), but it causes inconveniences for everyone else.</p> <p>Using <code>find</code> to scan the directories of other users (which you likely do when you start somewhere other than in your own directory) without their permission is considered anti-social behavior in our department, much like going through other people's desks or offices without their permission.</p> <div style="height: 15px; width: 10px;"></div> <p> Contact Us: <a href="mailto:software@cs.purdue.edu">Facilities Software Staff</a><br /> <span style="color: #777">Last Updated: September 08, 2010 09:29AM</span > </p> <div style="height: 15px; width: 10px;"></div> </div> <div id="info"> <div> <span class="header">Purdue University Policies</span> <a href="http://www.purdue.edu/policies/">Purdue Policies Homepage</a> <a href="http://www.purdue.edu/policies/records/viiia4.html">Purdue FERPA Policy</a> </div> <div> <span class="header">Purdue IT Policies</span> <a href="http://www.purdue.edu/policies/information-technology.html">IT Policies Home</a> <a href="http://www.purdue.edu/policies/information-technology/viib1.html">Authentication and Authorization (VII.B.1)</a> <a href="http://www.purdue.edu/policies/information-technology/viib2.html">Privacy for Electronic Information (VII.B.2)</a> <a href="http://www.purdue.edu/policies/information-technology/c-34.html">Data Security and Access Policy (C-34)</a> <a href="http://www.purdue.edu/policies/information-technology/c-42.html">Purdue Pages on the World Wide Web, Policy for (C-42)</a> <a href="http://www.purdue.edu/policies/information-technology/viia1.html">Electronic Mail (VII.A.1)</a> </div> </div> </div> <div style="clear: both;" /> <div id="footer"> <div style="font-size: 80%;font-weight: 800;float: left;"> Computer Sciences, 305 N.
