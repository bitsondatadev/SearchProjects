<HTML> <HEAD><TITLE>Help with PROLOG and PROGOL on the CS machines</TITLE></HEAD> <BODY BGCOLOR="ffffff"> </HEAD> <BR> <H2>PROLOG and PROGOL</H2> <HR NOSHADE SIZE=2> <P><!DOCTYPE HTML PUBLIC "-//IETF//DTD HTML//EN"></P> <ul> <li>PROLOG<BR> <BR> To experiment with PROLOG, you can use the XSB-PROLOG logic programming system available in /usr/local/xsb-prolog/xsb. You might want to make an alias to this as follows <tt><B>alias xsb "/usr/local/xsb-prolog/xsb -i"</B></tt>. An online manual is available <a href="http://www.cs.sunysb.edu/~sbprolog/xsb-page.html">here</a>. The best way to use this system is to insert all your facts into a file with a suffix (.P) and then load it as described below. Queries can be conducted once inside the interpreter. For example, if you insert the fact <tt><B>father(mary,william)</B></tt> into <tt><B>father.P</b></tt>, we have: <pre><B> UNIX Command prompt >xsb XSB Version 1.4.0 (94/5/9) [sequential, single word, optimal mode] | ?- [father]. [Compiling ./father] [father compiled, cpu time used: 1.389 seconds] [father loaded] yes | ?- father(william,mary). no | ?- father(mary,william). yes | ?- father(mary,X). X = william yes | ?- father(X,X). no | ?- father(X,Y). X = mary Y = william yes </B></pre> <hr> <li>PROGOL<BR> <BR> PROGOL is currently available as <tt><b>/homes/ramakris/PROGOL/progol</B></tt>. Some example PROGOL files are also available in the <tt><B>/homes/ramakris/PROGOL/examples</B></tt> directory. </ul> </P> <HR NOSHADE SIZE=2> </P> <A HREF="../index.html"><IMG SRC="../images/returnhome.gif" ALT="Return Home" BORDER=0></A> </BODY></HTML> </BODY></HTML>
