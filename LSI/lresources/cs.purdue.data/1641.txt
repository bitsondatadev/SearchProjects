Each tool must be from a different category listed in the leftmost column labeled &quot;Task.&quot; A tool not listed here may also be selected upon approval from the instructor. A consolidated list of open source software testing tools is available <a href="http://www.opensourcetesting.org/" target="_parent">here</a>.</p> <p>Download the tool, learn to use it and prepare a brief, at most 15 minutes, presentation. The presentation will be on Thursdays as per the schedule available<a href="http://www.cs.purdue.edu/homes/apm/courses/CS490SoftwareTestingSpring2011/videoConfSchedule.html" target="mainFrame"> here</a>. The presentation must include answers to the following questions:</p> <ol> <li>Who (company or individual) developed the tool?</li> <li>What testing purpose does the tool serve?</li> <li>What programming language(s) does the tool support, if any? </li> <li>In what phase of software tetsing is the tool useful?</li> <li>What are the strengths and limitations of the tool?</li> </ol>
