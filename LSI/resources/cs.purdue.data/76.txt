style="margin-top: 15px;" class="feature_title"> Remote access to CS Facilities </span> <p>This information is intended for those with authorized access to CS Department Facilities only.</p> <ul style="list-style-type: none;"> <li style="float: left;margin-left: 20px;"><a href="#RShell">Remote Shell Access</a></li> <li style="float: left;margin-left: 20px;"><a href="#REmail">Remote E-mail Access</a></li> <li style="float: left;margin-left: 20px;"><a href="#RShare">Remote Share Access</a></li> </ul> <br style="clear: both;" /> <a name="RShell" id="RShell" /> <h3>Remote Shell Access</h3> <p>The CS Department UNIX systems accept remote shell access to authorized users via SSH. When configured and used properly it should provide reasonable authentication and security. Our servers support SSH1 and SSH2, but strongly recommend SSH2.</p> <p>Modern Linux, MacOS X and UNIX systems have SSH client software installed.</p> <p>The recommended Windows client is Simon Tatham's <a href="http://www.chiark.greenend.org.uk/~sgtatham/putty/">PuTTY</a> suite of programs. If you want only the SSH client, you can save <a href="software/putty.exe">putty.exe</a> to your local disk and run it. Visit the <a href="http://www.chiark.greenend.org.uk/~sgtatham/putty/">PuTTY site</a> for the complete suite, documentation, etc.</p> <p>PuTTY saves host keys in the Windows registry and does not use host keys stored in a file. However, we do make our our <a href="ssh_known_hosts2">public host keys file</a> available for manual verification. You can also get the <a href="sshfingerprint.sxhtml">public key fingerprints</a>.</p> <a name="REmail" id="REmail" /> <h3>Remote E-Mail Access</h3> <p>ITaP provides documentation for their email services at <a href="http://www.itap.purdue.edu/email/">http://www.itap.purdue.edu/email/</a>.</p> <a name="RShare" id="RShare" /> <h3>Remote Share Access</h3> <p>Access to CS SMB shares (served by Windows machines or Samba servers on UNIX machines) is only possible through a campus VPN connection or SSH tunneling. Visit the <a href="http://www.itap.purdue.edu/connections/vpn/">Purdue VPN</a> page for information on setting up a VPN connection with a Windows machine.</p> <p>In general, access authentication is the domain name &quot;SERVERS&quot;, your CS login and your CS Windows password. How you specify the domain and login is dependent on your client -- typically with Windows it would be <b>SERVERS<i>login</i></b>. Linux clients may expect <b>SERVERS/<i>login</i></b> or <b><i>login</i>/SERVERS</b>.</p> <p>For instructions on using SSH to tunnel SMB mounts with Linux, see our <a href="http://support.cs.purdue.edu/help/FAQ,_UNIX#Can_I_mount_my_UNIX_directory_from_home_using_NFS.3F">UNIX FAQ</a>.</p> <p> <hr /> <a href="mailto:software@cs.purdue.edu"><em>Facilities Software Staff</em></a> </p> </div> </div> <div style="clear: both;" /> <div id="footer"> <div style="font-size: 80%;font-weight: 800;float: left;"> Computer Sciences, 305 N.
