<!dOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN"> <HTML><HEAD><TITLE>Xiangyu's Projects - Distributed Systems</TITLE> <META http-equiv=Content-Type content="text/html; charset=iso-8859-1"> <META content=xxx name=Author> <META content="MSHTML 6.00.2800.1264" name=GENERATOR></HEAD> <BODY text=#000000 vLink=#cc0000 aLink=#cc0000 link=#cc0000 background="figures/yellowbg.jpg"> <STYLE type=text/css>A { TEXT-DECORATION: none } </STYLE> <h2> Debugging Distributed Systems</h2> <img border=0 width=800 height=5 src="figures/line.bmp"> <p> In this project, we aim to develop debugging techniques for distributed systems and sensor networks as such systems become more and more pervasive. We focus on post-mortem analyses for runtime failures. </p> <p> Our contributions are highlighted as follows. <ul> <li> Different from regular programs, when debugging distributed systems, developers have to reason globally at different levels, from high level node-to-node communication to local instruction executions. A bug in one node may ripple through the system, causing a failure in another node. We have developed <font size=3><b>a unified query based debugging interface</b></font> for distributed systems. Information at different levels are modeled in relations and can be queried jointly in a unified way as many times as needed. Queries are answered by replaying and analyzing the execution <a href="Comp/dsn11.pdf">[DSN'11]</a>. </li> <li> Replay for distributed systems has the unique challenge of dealing with long executions, i.e. a failure may occur after a long period of normal execution, leading to long replay time. We proposed an <font size=3><b>execution reduction technique</b></font> that can generate reducible replay log. In particular, by analyzing the log alone, we can have a much smaller replay log that still produces the same failure. We leverage the compiler to enhance the replay log with necessary information to achieve reduction <a href="Comp/pldi11.pdf">[PLDI'11]</a>. </li> <li> In deployed distributed systems, execution occurs remotely without human intervention. Debugging such systems often relies on analyzing system logs. We developed <font size=3><b>a post-mortem log analysis to discover event causality</b></font>. The challenge lies in that one node may serve multiple requests simultaneously and thus the corresponding events are all mingled in the log file <a href="Comp/srds10.pdf">[SRDS'10]</a>. </li> <li> We proposed an efficient <font size=3><b>conrol flow tracing technique for sensor networks</b></font>. Small amount of instrumentations are added to sensor programs through compilers. Execution of the instrumentations generates precise encoding of control flow trace. In particular, an execution path, which may be interprocedural, can be encoded into one integer number <a href="Comp/sensys10.pdf">[ACM SENSYS'10]</a>. </li> </ul> </p> <h3>Funding</h3> <table> <tr height="40"> <td valign="top" height="40"><img align="left" width="42" height="35" BORDER=0 src="figures/nsf.gif" alt=""></a></td> <td height="40" valign="top"><a href="https://www.fastlane.nsf.gov/servlet/showaward?award=0834529"> <font face="Arial" size="2"> <i>A Holistic Approach to Reliable Pervasive Systems</i></a>, NSF-CSR-0834529, 2008-2011.</td> </table> <h3>Students</h3> <ul> <li> <a href="http://www.cs.purdue.edu/homes/lee612/">Kyu Hyung Lee</a> <br> </li> <li> <a href="http://web.ics.purdue.edu/~vsundar/">Vinai Sundaram</a> <br> </li> <li> Bin Xin (google)<br> </li> </ul> <h3>Publications</h3> <table> <tr> <td align="left" valign="top"> <font color="#660000" > <b>DSN</b></font></td> <td align="left" valign="top"> <b>K.
