Some people have been successful in porting it to other systems as well. </p> <p> The latest source version is RCS Version 5.8 from Thien-Thi Nguyen. It is available in <a href="rcs-5.8.tar.gz">gzipped tar format (1M). </p> <p> The latest PC (OS/2 DOS Win95 NT) binary version is available as three ZIP files. They are <a href="rcs57pc1.zip">rcs57pc1.zip (1.2MB)</a>, <a href="rcs57pc2.zip">rcs57pc2.zip (0.9MB)</a> and <a href="rcs57pc3.zip">rcs57pc3.zip (0.9MB)</a>. They contain everything you should need to use RCS on a PC. If you want to try to compile RCS on a PC, you will need the <a href="rcs57-pc.zip">supplementary source file (41K)</a>. This contains diffs relative to RCS 5.7 and diffutils 2.7.1 sources. </p> <p> If you have problems fetching the files with HTTP, you can use FTP to fetch the necessary files from the <a href="ftp://ftp.cs.purdue.edu/pub/RCS/">RCS FTP site</a>. </p> </div> <br /> <a name="DOC"></a> <h3>Documentation</h3> <div class="main"> <p> UNIX style man(ual) pages are included in the distributions. There is some additional documentation available on-line. The first is a modified version of the paper that appeared in <i>Software Practices &amp; Experience</i>, July 1985. It is available in <a href="rcs.ps">PostScript (138K)</a> or <a href="rcs.ms">*roff (55K)</a>. If you want to *roff format, but don't have the <b>pic</b> preprocessor, you need to the <a href="rcs.ms.nopic">pre-<b>pic</b>ed (63K)</a> version. </p> <p> The second document is a one-page summary describing the functions of RCS. It is available in <a href="rcs_func.ps">PostScript (9K)</a> or <a href="rcs_func.ms">*roff (3K)</a>. </p> <p> Aaron Hawley (<a href="mailto:&#65;&#97;&#114;&#111;&#110;&#46;&#72;&#97;&#119;&#108;&#101;&#121;&#64;&#117;&#118;&#109;&#46;&#101;&#100;&#117;">&#65;&#97;&#114;&#111;&#110;&#46;&#72;&#97;&#119;&#108;&#101;&#121;&#64;&#117;&#118;&#109;&#46;&#101;&#100;&#117;</a>) has made <a href="http://www.uvm.edu/~ashawley/rcs/tichy1985rcs.html">HTML and DocBook XML versions</a> of the SP&amp;E paper available online. </p> <p> The third document is a book written by Brian O'Donavan while at Digital Equipment Corporation. It was never published, but he gave it to Purdue to see if there was any interest from RCS users. It is a <a href="contrib/odonavan/RCS_Book.tar.Z">compressed tar file (248K)</a> </p> <p> O'Reilly & Associates, Inc. also published <a href="http://www.oreilly.com/catalog/rcs/"><u>Applying RCS and SCCS</u></a> by Don Bolinger and Tan Bronson. </p> </div> <br /> <a name="HELP"></a> <h3>Help</h3> <div class="main"> <p> Please see the <a href="FAQ.html">FAQ</a> as a first pass. If this does not answer your question, you can send mail to the <a href="mailto:&#104;&#101;&#108;&#112;&#45;&#114;&#99;&#115;&#64;&#103;&#110;&#117;&#46;&#111;&#114;&#103;">GNU RCS help mailing list</a> for general help with RCS, or the <a href="&#109;&#97;&#105;&#108;&#116;&#111;&#58;&#98;&#117;&#103;&#45;&#114;&#99;&#115;&#64;&#103;&#110;&#117;&#46;&#111;&#114;&#103;">GNU RCS bug report list</a> for bug reports. </p> <p> If you cannot get help anywhere else, you can send mail to the <a href="mailto:&#114;&#99;&#115;&#45;&#98;&#117;&#103;&#115;&#64;&#99;&#115;&#46;&#112;&#117;&#114;&#100;&#117;&#101;&#46;&#101;&#100;&#117;">RCS-BUGS</a></address> address. This is a mailing list of people that can either answer your question or direct it to the actual code developers. </p> </div> <br /> <a name="CONT"></a> <h3>Contributed<br />Software</h3> <div class="main"> <p> All contributed software is made available as-is, with no warranty by either the contributors or the RCS maintainers. </p> <ul> <li> Kenneth Cox contributed <a href="contrib/cox/sccs2rcs.csh">sccs2rcs</a>, a csh script to convert an existing SCCS history into an RCS history. It is the version distributed with CVS in the contrib section, as modified by Brian Berliner. </li> <li> Eric S. Raymond created a modified version of Kenneth Cox's sccs2rcs script. He has provided the <a href="contrib/raymond/sccs2rcs.csh">csh source</a> of the most recent csh version (1.3) which includes all the functionality of Ken Cox's and Brian Berliner's versions. He has also provided a python rewrite at <a href="http://www.catb.org/~esr/sccs2rcs/">http://www.catb.org/~esr/sccs2rcs/</a> </li> <li> Ray McVay contributed a perl version of <a href="contrib/mcvay/rcsfreeze.pl">rcsfreeze</a>. </li> </ul> </div> <br /> <a name="OTH"></a> <h3>Other Resources</h3> <div class="main"> <p> Please check out the <a href="http://www.gnu.org/software/rcs/rcs.html">GNU project web page for RCS</a>. </p> </div> <br /> <a name="REL"></a> <h3>Related Tools</h3> <div class="main"> <p> <a href="http://CVSHome.org/">CVS</a> is an open source version control system layered on top of RCS, designed to manage entire software projects. </p> <p> <a href="http://www.applied-cs-inc.com/">Applied Computer Sciences</a> offers their <a href="http://www.applied-cs-inc.com/bcintro.html">/BriefCase 3 Toolkit</a> for free under the GPL, and promotes it as a true client/server solution for UNIX, providing better project management than CVS. </p> <p> <a href="http://www.xcf.berkeley.edu/~jmacd/prcs.html">PRCS</a> is supposed to be a much simpler CVS-like tool. </p> <p> <a href="http://www.bitmover.com/bitkeeper/">BitKeeper</a> is a fully distributed source management system from BitMover, Inc. It is based on their version of SCCS and is file format compatible with AT&amp;T SCCS. </p> <p> <a href="http://www.canb.auug.org.au/~millerp/aegis.html">Aegis</a> is a transaction-based software configuration management system. </p> </div> <br /> <hr> <address><a href="mailto:&#114;&#99;&#115;&#45;&#98;&#117;&#103;&#115;&#64;&#99;&#115;&#46;&#112;&#117;&#114;&#100;&#117;&#101;&#46;&#101;&#100;&#117;">RCS Maintainers</a></address> <!-- Created: Tue Mar 31 16:06:56 EST 1998 --> <!-- hhmts start --> Last modified: Fri Oct 14 13:48:18 EDT 2011 <!-- hhmts end --> </body> </html> </html>
