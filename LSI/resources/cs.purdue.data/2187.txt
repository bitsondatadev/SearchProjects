PELLPACK consists of an interactive graphical interface which supports symbolic processing, equation editors, PDE model specification, geometry tools, analysis tools, mesh generators, geometry partioning tools, visualization packages, PDE solving libraries, integrated foreign systems, machine configuration, and parallel processing. <br clear=left> <a href="images/map.gif"> <img align=right src="images/map-small.gif" hspace=8 vspace=3></a> A web-based demonstration has been established at <a href="http://www.cs.purdue.edu/research/cse/pellweb/">WebPDELab</a> For an overview of the web project, click <a href="http://www.cs.purdue.edu/research/cse/hpc-pses/pellweb.html">here</a>. The PELLPACK network server provides a powerful internet based problem solving facility. Users anywhere on the internet can access it to solve a broad range of simulations for physical objects and processes. <br clear=right> <b>Click <a href="papers/pellpack.ps">here</a> to read the most comprehensive paper written about the PELLPACK project. </b> [1mb ps file] (*) <HR SIZE=4> <!-- PYTHIA-II --> <a href="http://www.cs.purdue.edu/research/cse/ifestos/index.html"> <h2> The PYTHIA-II Project</a></h2> Data Mining and Knowledge Discovery in Databases (KDD) are emerging areas of research that include statistical analysis, databases, pattern matching/rules generation, and high performance computing. KDD is the overall process of discovering valid, useful, understandable knowledge in databases, while Data Mining is one phase of that process. Data Mining includes the application of statistical techniques to data, the use of algorithms to extract structures, and the enumeration of patterns or models that explain the behavior of the data. <P> <a href="images/pythia.gif"> <img align=left src="images/pythia-small.gif" hspace=8 vspace=3> </a> PYTHIA-II applies KDD to the study of scientific software for the solution of PDEs. The result of PYTHIA-II research is a Recommender system which answers user queries regarding the applicability, efficiency and accuracy of software for solving PDE problems. Supported by an inference process, the PYTHIA-II Recommender assimilates information from knowledge bases built by the data mining of domain specific performance data. Once the computational objectives for the performance evaluation of the selected scientific software have been determined, the KDD process is used to generate a knowledge base as shown in the table below. The implementation of PYTHIA-II includes a complete graphical user interface supported by the POSTGRES95 RDBMS, in-house statistical software, the GOLEM ILP learning system and the CLIPS expert system tool-box. <br clear=left> <P> <table border=2> <tr> <td> Data <br> Preparation</td> <td> select benchmark applications<br> (problem population) </td> <td> identify performance measures </td> <td> select methods <BR>(libraries/routines & parameters) </td> <td> use domain-specific schema <BR>to populate database </td> </tr> <tr> <td> Data <br> Generation</td> <td> use database population <BR> to build PDE 'programs' </td> <td> execute programs, generating performance output </td> <td> collect and store performance data </td> <td> &nbsp </td> </tr> <tr> <td> Data <br> Mining </td> <td> prepare/extract/filter/clean data </td> <td> apply statistical techniques </td> <td> generate profiles to characterize methods and parameters </td> <td> identify patterns/models from analyzed data </td> </tr> <caption align=top> Steps of the KDD Process for Scientific Software </caption> </table> <P> Look at the preliminary paper <a href="http://www.cs.purdue.edu/research/cse/pythia-II/paper.ps">PYTHIA-II: A KDD Based Recommender System for Scientific Computing</a> (8MB postscript file) for more information about PYTHIA-II. <hr size=4> <!-- PDELAB --> <h2> <a href="http://www.cs.purdue.edu/research/cse/pdelabPPK/pdelab.html"> The PDELAB Project</a></h2> <a href="images/pdelab1.gif"> <img align=left src="images/pdelab1-small.gif" hspace=8 vspace=5></a> <a href="http://www.cs.purdue.edu/research/cse/pdelabPPK/pdelab.html"> <b>PDELab</b></a> is a problem solving and development environment for PDE based applications. PDELAB introduces the <b>PDESpec</b> symbolic language which invokes Mathematica for symbolic processing. <br clear=left> <a href="images/pdelab.gif" hspace=8 vspace=5> <img align=right src="images/pdelab-small.gif"></a> The PDELAB Notebook, Object Manager, Editor Toolkits and PDE solver components use PDE object representations as their foundation, and the PDELAB environment is implemented using the Purdue PSE Kernel (PPK). <br clear=right> <a href="images/pdelab2.gif"> <img align=left src="images/pdelab2-small.gif" hspace=8 vspace=5></a> The PDELAB components can be wired together at a high level to form a custom PDE solver for a given physical application. PDELAB allows users to introduce new PDE solver modules and build high level templates to study the behavior of solution processes. <br clear=left> <p> <hr size=4> <!-- SOFTLAB --> <h2> <a href= "http://www.cs.purdue.edu/research/cse/softlab/softlab-vlabs/softlab-vlabs.html"> The SOFTLAB Virtual Laboratories</a></h2> <a href="images/biolab.gif"> <img align=right src="images/biolab-small.gif" hspace=8 vspace=3></a> <b>SoftLab</b> is a framework for building virtual laboratories for computational science. A virtual laboratory is a hybrid PSE which encompasses all research activities that occur in the scientific laboratory environment. It is a software layer above the experimental and computational processes which generally exist in the laboratory side-by-side. The virtual laboratory unites the experimental and computational models in a way that allows them to interact with each other, so that feedback from one can enhance and improve the methodologies applied in the other. <br clear=right> <P> <hr size=4> View these <a href="http://www.cs.purdue.edu/research/cse/hpc-pses"> <b>slide presentations</b> </a> from a PSE review meeting at Purdue University in November, 1994. <p> Visit the <a href="http://www.cs.purdue.edu/research/cse/pses"> <b>Problem Solving Environments Research</B></a> homepage. <p> <hr size=4> <p> (*) If your web server cannot handle a postscript file, use <b>shift+button1</b> to download the file.
