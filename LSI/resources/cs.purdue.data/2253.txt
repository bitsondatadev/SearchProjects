It is believed to be Y2K compliant. However, because it is distributed in source form, all users are encouraged to examine the source to make sure it complies with their needs. <p> RCS version 5.5 and later stored dates with four digit years, but recognised 2 digit years in older RCS files. These versions should be Y2K compliant, but is it suggested that users upgrade to the latest version. <p> You can determine the version of RCS with the <b>-V</b> option to any of the commands: <font size="-1"><pre> % rcs -V RCS version 5.7 </pre></font> If your RCS does not support the <b>-V</b> option, it is probably too old to be Y2K compliant. </p> <a class="top" href="#TOP">TOP</a> </div> <a name="Q2"></a> <h3>2.</h3> <div class="main"> <b> During the install, I get the following error message </b> <font size="-1"><pre> ./conf.sh: testing permissions ... ./conf.sh: This command should not be run with superuser permissions. </pre></font> <p> The RCS configuration and build must be done as a non-root user. The reason is that some systems do not produce meaningful permission check results as root. In general UNIX applications should be configured and built as a non-root user, then installed (if being installed in a standard system directory) as root. </p> <a class="top" href="#TOP">TOP</a> </div> <a name="Q3"></a> <h3>3.</h3> <div class="main"> <b> Is there an easy way to convert SCCS files to RCS format? </b> <p> A script was posted years avo to the <code>comp.sources.misc</code> newsgroup to do just that. We have never used it, but others have asked for it and never reported problems. It is available as a <a href="SCCStoRCS.shar">shell archive as orignially posted</a>. </p> <a class="top" href="#TOP">TOP</a> </div> <a name="Q4"></a> <h3>4.</h3> <div class="main"> <b> Where do I get the GNU diffutils package? </b> <p> You can get information at the <a href="http://www.gnu.org/software/software.html">GNU project software page</a> </p> <a class="top" href="#TOP">TOP</a> </div> <a name="Q5"></a> <h3>5.</h3> <div class="main"> <b> How do I get <b>rlog</b> to display the correct time? </b> <p> <b>Rlog</b> is displaying the correct time, but it is displaying it as GMT, not local time. All times are stored in the RCS file as GMT. If you want times displayed in the local time, use the <b>-zLT</b> option, or see <b>rlog</b>(1) for more details. </p> <a class="top" href="#TOP">TOP</a> </div> <a name="Q6"></a> <h3>6.</h3> <div class="main"> <b> How do I get RCS to work with sshfs? </b> <p> This workaround was provided by Christian Delfosse. </p> <p> The problem is that sshfs does not allow renaming an existing file if the target name exists and is read-only. Enabling <b>bad_b_rename</b> in the RCS config causes RCS to first unlink the target, then do the rename. </p> <p> After running <b>configure</b> and <b>make</b>, manually edit src/conf.h and change <b>bad_b_rename</b> setting to 1. Then run <b>make</b> again. </p> <p> Miklos Szeredi with the sshfs team offered another workaround of using the <b>-oworkaround=rename</b> sshfs option. He also reports that this will be fixed in an upcoming release (already in the CVS respository) if you are using OpenSSH-4.9. </p> <a class="top" href="#TOP">TOP</a> </div> <br /> <h3></h3> <div class="main"> <p> For more information on RCS, see the <a href="index.html">Official RCS Homepage</a> </p> </div> <br /> <hr> <address><a href="mailto:&#114;&#99;&#115;&#45;&#98;&#117;&#103;&#115;&#64;&#99;&#115;&#46;&#112;&#117;&#114;&#100;&#117;&#101;&#46;&#101;&#100;&#117;">RCS Maintainers</a></address> <!-- Created: Tue Mar 31 17:19:49 EST 1998 --> <!-- hhmts start --> Last modified: Wed Jan 19 15:39:07 EST 2011 <!-- hhmts end --> </body> </html> </html>
